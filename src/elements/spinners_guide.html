Guidelines

When to Use

  • To indicate to the user that loading is occurring in the background.

When NOT to Use

  • When a loading scenario needs to indicate progress, usually with particularly long load times. Use a dynamically updating progress indicator instead.
  • For a main loading state, such as an app initialization, forego our simple spinner and opt to use a custom, branded one instead.

Visual Language

To inform judgment about color, type, space, and more
  • A spinner should usually be centered within its container unless inside of another element like a button.

Behaviors

To cover how to cope with events, outcomes, and transitions
  • A spinner spins indefinitely until loading is complete.
