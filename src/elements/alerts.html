Alerts & Toasts

Alerts contextual feedback messages for typical user actions with the handful of available and flexible alert messages. Toasts are temporary popup messages such as success message for save.

Differences

Toasts

Low priority.
User Action
Disappear automatically.
UI Specs
Floating Elements, have shadows.
Informational
Provide updates on app's processes
Temporary
Appear temporarily with a short message and mostly non-interractive.
Contextual
Placed in the most suitable area of the UI.
Alerts
Priority
Prominent, medium priority.
User Action
Remain until dismissed by the user, or if the state that caused the banner is resolved.
UI Specs
Inline Elements, no shadows.
Appropriately interruptive
Their level of interruption should match the information they contain and the context in which they appear.
Clear
Communicate a succinct message and what will happen if users interact with them.
Focused
Contain a single message and specific actions a user can take.

Alerts Code Structure & Usage

Alerts are available for any length of text, as well as an optional dismiss button. For proper styling, use one of the eight required contextual classes (e.g., .alert-success). For inline dismissal, use the alerts jQuery plugin.

Add class="alert" along with contextual classes and role="alert" to a div to create an alert.

Alerts with Buttons

You can add a button to an alert using any button style. For best results with single line alerts, use .btn-sm.

In these examples, we use the .float-right class to pull the button to the right and .clearfix on the alert to make sure the button is inside the alert correctly.

Dismissible Alerts

Add class="alert-dismissible fade show" to the alert and add a button with attributes class="close" data-dismiss="alert" to add dismissible alerts
